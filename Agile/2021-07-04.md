# 애자일 프로젝트 개요

- 애자일이란 : 신속하고 유연하며 적응적 개발을 목표로 함
  - 다양한 경량 개발 방법론 전체를 일컫음
- 증분 : 프로젝트 산출물을 빠르게 인도함
- 반복함으로서 리스크를 최소화함

- 애자일 방법론의 공통점
  - 고객이 프로젝트에 적극 참여함
  - 프로젝트의 리스크를 관리함
  - 역동적 일정 관리에 초점을 맞춤
  - 반복 기반 프로세스 활용
  - 문서보다 대화사용
  - 간단하고 속기로 기능 설명



- IT프로젝트가 실패하는 방법

광고/제안서/고객의 요구사항/고객에게 청구하는 금액

프로젝트 관리자의 이해 / 디자이너의 해석 / 개발자의 코드 / 베타테스트가 받은 것

산출물 문서화 / 고객이 정말 원하는 것 혹은 경쟁사 제품 / 오픈소스 제품 / 실제 사용

출시시기 / 패치적용 / 비상계획(재해복구대책) / 운영 부서의 지원



요구사항/기대사항을 구분해야 함
